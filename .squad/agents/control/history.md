## Learnings

### 2026-03-15: Team Setup & Security Baseline
- Two capabilities only: `fs:allow-write-file` (singular ‚Äî note the exact name) and `system-idle:allow-get-idle-time`
- CSP: WebView2 content locked to `tauri://localhost` only
- One credential in the entire system: Postgres/Supabase connection URI ‚Üí lives in OS keychain (keyring crate), never plain text
- Path traversal: all screenshot paths go through `std::fs::canonicalize` before write, then validated against configured screenshots directory
- No PII in logs: deny-listed process names must not appear in any log field, including window title fields
- No local HTTP/TCP listeners ‚Äî `health_get` is IPC-only, not a network endpoint
- Screenshots: never in sync queue, never transmitted. This is both a privacy and security requirement.
- Reviewer gate: I must sign off on all PRs touching capabilities, IPC handlers, file system, external network, or credentials

### 2026-03-15: T007 ‚Äî Capabilities Baseline Locked

**Verified permission identifiers (direct source confirmation):**
- `core:default` ‚Äî valid Tauri 2 built-in permission; confirmed as `const` enum value in `gen/schemas/desktop-schema.json` line 1982
- `fs:allow-write-file` ‚Äî confirmed in `tauri-plugin-fs-2.4.5/permissions/autogenerated/commands/write_file.toml`; grants `write_file`, `open`, `write` commands only (singular set, not the broader `fs:write-files` which would also grant create/copy/remove/rename/truncate)

**Permission explicitly NOT included:**
- `system-idle:allow-get-idle-time` ‚Äî plugin does not exist on crates.io; tasks.md T007 spec is stale on this point; idle is Win32 direct (`GetLastInputInfo` + `GetTickCount64`)

**CSP status:**
- Current: `default-src 'self' tauri: asset: https://asset.localhost` ‚Äî acceptable for dev phase
- Production tightening required in Final Phase: add `connect-src 'none'`; audit whether `asset:` is still needed after Blazor WASM compilation
- Architecture advantage: sync (T070) is Rust-only ‚Äî WebView2 never touches Supabase directly ‚Äî CSP does NOT need a `connect-src` exemption for sync

**Scope of `windows: ["main"]`:**
- Single named window, no glob. Correct and minimal.

**Future capability flags:**

**Path traversal reminder (standing requirement):**
